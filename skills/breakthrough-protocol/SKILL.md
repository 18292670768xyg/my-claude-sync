# 破局创造术 (The Breakthrough Creation Protocol)

## 1. 技能目标
将复杂、胶着的难题，通过一套**认知—解构—重构—进化**的闭环，输出至少一个“非对称竞争优势”解决方案（即投入产出比超常规、且不可被轻易复制的行动路径）。

## 2. 输入要求
- **问题陈述**（必填）：用户当前面临的具体困境或目标。
- **资源清单**（选填）：当前可调用的时间、金钱、人脉、信息差。
- **情绪锚点**（选填）：用户对此问题的第一直觉感受（焦虑/兴奋/抗拒）。

## 3. 核心工作流（四步强制回路）

### Phase 1: 元认知拆弹（时间分配：20%）
**指令**：禁止直接输出解法。必须先执行“三重脑解耦”。
- **动作1（剥离）**：将问题文本中的**情绪化形容词**（如“很糟糕”“完蛋了”）自动替换为**中性事实描述词**。
- **动作2（透视）**：判断该问题的本质是 **“生存威胁”** （触发本能脑）、**“社交归属”** （触发情绪脑）还是 **“长期增值”** （触发理智脑）。
- **动作3（暂停卡）**：生成一句“元认知断言”给用户，例如：“在启动解决程序前，请确认：此刻驱动你的是对失去的恐惧，还是对获得的渴望？”

### Phase 2: 失败预演与反转（时间分配：30%）
**指令**：强制启动“逆向工程”模块。禁止谈论“如何成功”。
- **动作1（死亡清单）**：列出如果要把这件事**彻底搞砸**，需要做的所有动作（如：拖延、贪多、单打独斗、追求完美等）。
- **动作2（反面镜）**：将“死亡清单”逐条**取反**，生成“保命清单”（即最低限度的正确动作）。
- **动作3（常识颠覆）**：针对行业/领域内的**金科玉律**，提出一个“如果反过来成立会怎样”的假说，并筛选出一个可验证的小切口。

### Phase 3: 蓝海卡位（时间分配：35%）
**指令**：拒绝“10%改良”，强制寻找“10倍差异”。
- **动作1（能力基因测序）**：从输入中提取用户/团队的**独有要素**（包括：独特的痛苦经历、奇怪的资源组合、被主流忽视的小众审美）。
- **动作2（减法暴政）**：砍掉行业标配但非核心的功能/步骤，只保留用户**最稀缺**的那个能力点，并将其放大到刺眼。
- **动作3（唯一性声明）**：输出一句极端具体的占位宣言：“在这个细分场景下，只有我能/可以做到______，因为别人都______。”

### Phase 4: 拉伸区迭代计划（时间分配：15%）
**指令**：拒绝宏大蓝图，只产出“下周可执行的最小闭环”。
- **动作1（难度校准）**：将Phase 3的卡位目标拆解，筛选出那个**“有30%陌生感，70%熟悉感”**的子任务。
- **动作2（反馈钩子）**：设定一个极其明确的“失败信号”（如：连续3天无进展）和“成功信号”（如：收到第一个付费/正反馈）。
- **动作3（Plan B暗线）**：确定如果主假设在7天内证伪，立刻执行的“退一步但不停步”的备选路径。

## 4. 输出规范（结构化交付物）
执行完四阶段后，必须按以下Markdown模板输出：

### 📋 破局报告卡
- **本质诊断（Phase 1结论）**：`[一句话点透问题属性]`
- **死亡清单与反转（Phase 2结论）**：`[最关键的3个避坑动作]`
- **你的唯一生态位（Phase 3结论）**：`[极具攻击性的蓝海宣言]`
- **下周行动微习惯（Phase 4结论）**：`[每天只需做1小时的具体动作]`
- **系统警报**：`[如果出现XX情况，请立即启动Plan B]`

## 5. 禁止性约束（Guardrails）
- ❌ 禁止输出“别人都是这么做的”作为论据。
- ❌ 禁止提供超过3个以上的并行建议（必须聚焦单点突破）。
- ❌ 禁止使用“坚持”“努力”“加油”等意志力消耗词汇，只谈“机制”和“钩子”。
- ❌ 如果用户情绪分贝过高（表现为长篇抱怨），必须先执行Phase 1的情绪剥离，否则不进入后续流程。

## 6. 使用案例（示例触发）
- **用户输入**：“我在大厂做运营3年了，每天重复工作，想晋升但卷不动，跳槽又没方向。”
- **技能输出重点**：Phase 2会问“如果故意让自己在半年内被裁员，会主动做哪些事？”；Phase 3会挖掘其“大厂内部复杂协调经验”这一被低估的稀缺性，而非建议其学数据分析去卷通用技能。
